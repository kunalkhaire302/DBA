Section 2 - Problem Statement

In modern banking, maintaining vast volumes of financial records requires more than just storing data. A bank needs stringent Database Administration, security, and monitoring to prevent data loss, ensure atomic financial transactions, and maintain confidentiality. This project mitigates common banking risks—such as unauthorized access, slow querying, and system failure—by implementing an enterprise-grade Oracle-style database infrastructure. It enforces Role-Based Access Controls, deploys automated auditing triggers, and visualizes strict query optimizations to guarantee operational integrity and seamless disaster recovery management.

Section 3 - Objectives

  • Design a highly normalized ER model to remove data redundancy.
  • Ensure ACID compliance on all fund transfers using transactional SQL logic.
  • Implement Role-Based Access Control to separate Teller and Administrator capabilities.
  • Deploy database triggers to maintain an immutable Audit Log of all mutations.
  • Utilize Views and Indexing to accelerate reporting performance.
  • Provide real-time telemetric monitoring for query tracking and system health.
